## 팀

<br><br>

### 멘토 지원 받기

<br>

- 팀별 멘토 지원 (기존 멤버들 + 새로 오신 분들 중 하고 싶으신 분들)
- 팀별 멘토 공부 방향이나 질문 등을 받아주고 도와주는 역할. 그렇다고 뭐 엄청 대단한 사람은 아님.
- 팀별 멘토 또한 물론 다른 팀에 들어가 구성원 역할을 수행할 수 있음.
- 팀별로 어떻게 공부를 해 나가면 되는지 이야기
- 없으면 끝.

<br><br>

### 팀 짜기

<br>

- 전체적인 팀 설명 후 자유롭게 돌아다니며 팀 멘토에게 Q&A 시간 갖기
- Q&A가 끝난 후 최우선 1개의 팀으로 모이기
- 한 학기동안 유지될 팀입니다.
- 하지만 다른 팀의 스터디는 얼마든지 들으실 수 있습니다.
- 자, 이제 자신의 소개 종이를 들고 여기저기 돌아다니면서 관심분야에 대해 이야기해보도록 합시다.

<br><br>

### 팀별 공부내용, 목표량, 미팅시간, 방법 정하기

<br>

- 자 이제 팀별로 모여 주세요!
- 팀끼리 한번 더 자기소개서를 공유하면서 이야기나누어보도록 합시다.

- **팀의 구성 : 그룹장, 멘토, 구성원**

- 팀별 멘토 : 기존 멤버와 새로 오신 분들 중, '잘 하시는 분' 으로 방향성을 도와주는 역할
- 팀별 그룹장 : 팀원의 출석과 공부방향을 함께 논의 및 의결, github 에 작성하는 역할
